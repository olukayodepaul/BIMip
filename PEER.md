## ü§ù Peer

The **`Peer`** message contains server-assigned **stream synchronization metadata** critical for reliability. It includes the base IDs of both the sender and the recipient, along with the global offsets generated by the server for stream management.

### Protobuf Definition

```protobuf
message Peer {
  string from = 1;
  string to = 2;
  int64 offset = 3;
  int64 peer_offset = 4;
}
```

-----

### Field Reference

| Tag | Field Name | Type | Description | Primary Role |
| :---: | :--- | :--- | :--- | :--- |
| **1** | `from` | `string` | The **base ID** (`eid`) of the message sender in the context of the stream. | Context |
| **2** | `to` | `string` | The **base ID** (`eid`) of the message recipient in the context of the stream. | Context |
| **3** | `offset` | `int64` | The **server's global stream offset** assigned to the message this `Peer` object is associated with. This identifies the message's position in the master stream. | **Message Position** |
| **4** | `peer_offset` | `int64` | The **recipient's stream offset** as known and tracked by the server. This represents the last known position of the recipient in their consumption stream. | **Recipient Stream State** |
